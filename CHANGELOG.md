# Changelog

## v0.15.0

- Добавлен единый манифест обязательных зависимостей агента для `apk` и `opkg`; он применяется после установки и каждого обновления.
- `nlbwmon` стал обязательной зависимостью: агент проверяет UCI, автозапуск, службу и локальный запрос счётчиков, а при сбое перезапускает службу.
- В telemetry добавлены скорость, duplex, MAC, состояние, RX/TX, пакеты, ошибки и drops физических сетевых устройств.
- WAN-форма Web UI показывает только поля выбранного протокола; Android сохраняет тот же принцип.
- Добавлено управление DNS over TLS через Stubby и DNS over HTTPS через `https-dns-proxy`, включая установку пакетов из Web UI и Android.
- Диагностика агента публикует полный статус обязательных зависимостей.

## v0.14.2

- Исправлена интеграция `nlbwmon`: служба включается после установки, состояние выборки передаётся явно, CSV разбирается по именам колонок.
- Web UI получил быстрый поиск по списку установленных пакетов.
- Агент передаёт реальные `ip6assign`, `ip6hint`, RA, DHCPv6 и NDP для LAN.
- Управление IPv6 домашней сети перенесено из WAN в раздел клиентов и синхронизировано между Web UI и Android.
- Android `versionCode` повышен до `58`; capability schema остаётся `13`.

## v0.14.1

- Статус присутствия клиентов разделён на подтверждённый, недавний и отключённый.
- DHCP-аренды, `PERMANENT` и `NOARP` исключены из доказательств активности; повторный `STALE` больше не продлевает статус бесконечно.
- Wi-Fi station, активные neighbour-состояния и рост RX/TX подтверждают присутствие; при остановке агента статус автоматически протухает.
- API, Web UI и Android показывают единые счётчики, фильтры, источник и время последнего подтверждения.
- Добавлена миграция PostgreSQL `0008_client_presence`; Android `versionCode` повышен до `57`.
- Метаданные OpenWrt-агента синхронизированы с `0.14.1`, capability schema остаётся `13`.

## v0.14.0

- Добавлены одноразовые mobile pairing tokens: срок 10 минут, однократный обмен, ручной отзыв и PostgreSQL rate limiting.
- В базе хранится только SHA-256 hash pairing token; исходные pairing/access/refresh tokens не попадают в аудит.
- Web UI получил раздел подключения Android с QR, каноническим адресом сервера, обратным отсчётом и отзывом кода.
- Сессии получили тип клиента; мобильные pairing-сессии видны владельцу и отзываются отдельно.
- Android получил CameraX/ML Kit QR-сканер, строгую проверку формата v1, подтверждение домена и понятные состояния expired/used/revoked.
- Удалён небезопасный plaintext fallback `SharedPreferences`; сессии сохраняются только через `EncryptedSharedPreferences`.
- HTTP теперь разрешается только для локальных адресов при `WRTMONITOR_ALLOW_INSECURE_LOCAL=true`.
- Добавлена миграция PostgreSQL `0007_mobile_pairing`; Android `versionCode` повышен до `56`.
- Метаданные OpenWrt-агента синхронизированы с `0.14.0`, capability schema остаётся `13`.

## v0.13.7

- Оперативные показатели считают только клиентов, которые действительно присутствуют в ARP/ND или hostapd; сохранённые DHCP-аренды остаются в истории, но больше не выдаются за подключённые устройства.
- Обзор, системный экран, Android и история telemetry используют единый показатель активных клиентов.
- Карточка Wi-Fi теперь отдельно показывает число радиомодулей и число клиентов в эфире.
- Метрики hostapd нормализованы: сигнал, скорость RX/TX, длительность подключения и airtime RX/TX выводятся только при наличии данных драйвера.
- Сырые объекты airtime больше не попадают в Web UI.
- Колонка трафика клиентов скрыта, если nlbwmon не подтвердил рабочую базу или ещё не передал счётчики; интерфейс показывает точную причину.
- Разбор таблицы соседей агента больше не зависит от фиксированного числа полей `ip neigh`.
- Android `versionCode` повышен до `55`.

- Главный экран роутера очищен от диагностики, команд, capabilities и raw telemetry.
- Добавлены новый список роутеров и компактная история команд.
- Android получил обзор состояния сети; управление агентом перенесено в раздел системы.
- Исправлено сохранение текущего раздела Web UI после отправки команды.

## v0.2.0-rc1

- Введена telemetry schema v2 с нормализованными блоками системы, сервисов, Wi-Fi, сети и клиентов.
- Добавлены DHCP leases, neighbour table, conntrack, kernel, hostname, load 1/5/15 и состояние ключевых OpenWrt-сервисов.
- Добавлено управление Wi-Fi каналом и регионом.
- Добавлены переподключение интерфейса и перезапуск сети.
- Добавлены изменение hostname и перезапуск сервисов из allowlist.
- Добавлено управление статическими DHCP-выдачами.
- Web UI и Android синхронно расширены новыми экранами и командами.
- Конфигурационные команды защищены валидацией, capability checks, подтверждением, аудитом и backup.
- FastAPI startup переведён с deprecated `on_event` на lifespan.

## v0.1.1-rc9-agent-modularization-and-ui-fixes

- OpenWrt-агент переведён на модульную структуру `wrtmonitor-agent + lib/*.sh`.
- Installer и update pipeline обновлены под manifest `openwrt-agent-files.txt` и новую схему установки `/usr/lib/wrtmonitor`.
- Сервер, Android и GitHub Actions переведены на единый источник версии через `VERSION`, `VERSION_CODE` и `RELEASE_TAG`.
- Android переведён на `EncryptedSharedPreferences` для хранения сессии и access token.
- Из Android вычищены устаревшие formatter/helper хвосты и добита локализация рабочих экранов.
- Backend явно закреплён как `single-owner` модель доступа и отражает это через `/health/config`.
- Web UI получил compact capabilities UX и явное сообщение о необходимости reinstall `rc9`, если агент ещё не передал capabilities.
- Android получил такой же compact capabilities UX на экране устройства.
- Удаление из активного списка закреплено только для `disabled` устройств в Web UI и Android.

## v0.1.1-rc8-router-management-core

- Добавлены `agent capabilities`, нормализованные `agent/wifi/network` блоки telemetry и endpoint `GET /api/v1/devices/{device_id}/agent`.
