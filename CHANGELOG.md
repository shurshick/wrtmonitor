# Changelog

## v0.10.4

- Исправлена диагностика Web UI-входа через HTTPS reverse proxy и локальный HTTP.
- Вместо тихого возврата на форму показывается причина блокировки защищённой cookie.
- Добавлена ссылка на настроенный публичный HTTPS-адрес.
- Логин очищается от случайных пробелов в Web UI и Android API.
- Android `versionCode` повышен до 45.

## v0.10.3

- Журнал команд ограничен пятью записями на страницу и получил навигацию.
- Завершённые команды автоматически очищаются через 30 дней или после превышения лимита 500 записей на роутер.
- Активные команды очистка не затрагивает; срок и лимит настраиваются через окружение.
- История скачиваемых backup и diagnostics больше не зависит от текущей страницы журнала.
- Android `versionCode` повышен до 44.

## v0.10.2

- Исправлена передача фактического LAN IPv4/префикса/маски, DHCP-пула и настроек времени из OpenWrt.
- Web UI и Android больше не подставляют демонстрационные сетевые параметры вместо конфигурации роутера.
- Параметры с конечным набором значений переведены на списки выбора.
- Убраны фиксированные firewall-зоны Android и адрес гостевой сети агента.
- Исправлена нормализация нового и старого форматов network telemetry; Android `versionCode` повышен до 43.

## v0.10.1

- Исправлено числовое сравнение stable/RC версий агента на границе `0.9 -> 0.10`.
- Добавлен безопасный compatibility-переход с агента `0.9.0` через обычную кнопку обновления.
- Добавлены регрессионные тесты semver comparator; Android `versionCode` повышен до 42.

## v0.10.0

- Добавлены управляемые refresh-сессии с одноразовой ротацией, отзывом и 15-минутным access-token.
- Добавлены смена пароля владельца, активные сессии, журнал аудита и эксплуатационные уведомления в Web UI и Android.
- Реализованы PostgreSQL backup, проверка архива, restore drill во временную БД и контролируемое восстановление.
- TrueNAS получил отдельный постоянный volume резервных копий.
- Релизный Android APK подписывается закрытым production-ключом из GitHub Secrets; `versionCode` повышен до 41.
- CI дополнен disaster-recovery и проверкой подписи Android.

## v0.9.0

- Добавлены пакеты `opkg`, backup/restore, проверяемый sysupgrade, журналы, процессы и cron.
- Добавлены скачиваемый диагностический архив и ограниченный recovery mode.
- Web UI и Android получили единый capability-aware раздел обслуживания роутера.
- Защищено удаление системных пакетов, прошивка проверяется по HTTPS, SHA-256, модели, месту и совместимости.
- Многострочные результаты агента теперь корректно экранируются в JSON; большие архивы не попадают в историю команд.
- Capability schema повышена до 10, Android `versionCode` — до 40.

## v0.8.0

- Добавлено управление WireGuard-интерфейсами и peer: адреса, ключи, endpoint, AllowedIPs, keepalive, импорт и безопасный экспорт конфигурации.
- Добавлены OpenVPN client profiles и policy-based routing по клиентам, подсетям и назначениям.
- VPN-телеметрия показывает туннели, handshake, RX/TX, состояние OpenVPN и политики PBR.
- VPN-команды включены в config transaction, backup и rollback; секреты скрыты в API, preview и журнале команд.
- Web UI и Android получили capability-aware VPN-раздел; capability schema повышена до 9.
- Android `versionCode` увеличен до 39.

## v0.7.0

- Добавлены IPv6, RA и DHCPv6 для локальных интерфейсов.
- Добавлены Multi-WAN/failover, статические IPv4/IPv6-маршруты и DDNS.
- Реализовано управление UPnP, зонами, транзитом и правилами firewall.
- Perimeter-телеметрия показывает маршруты, зоны, правила, DDNS, состояние `mwan3` и активные UPnP mappings.
- Web UI и Android получили одинаковые capability-aware действия; schema capabilities повышена до 8.
- Конфигурационные команды включены в транзакции с backup, проверкой связи и rollback соответствующих служб.

## v0.6.0

- Добавлено управление несколькими SSID на каждом радиомодуле: создание, изменение и удаление.
- Радиопараметры объединены в безопасную команду: канал, ширина, регион и мощность.
- Добавлены расписания Wi-Fi с поддержкой интервалов через полночь.
- Добавлены client isolation, 802.11r/k/v и Mesh 802.11s там, где их поддерживает OpenWrt.
- Телеметрия Wi-Fi дополнена signal, noise, bitrate, airtime и временем подключения клиентов.
- Web UI и Android получили одинаковые новые Wi-Fi-действия; capability schema повышена до 7.
