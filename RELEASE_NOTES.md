# v0.2.0-rc2

Исправляющий релиз интерфейсов. `rc1` расширил backend и команды, но оставил Web UI и Android в виде перегруженных технических экранов. В `rc2` изменена именно структура пользовательского интерфейса.

## Что изменилось

- Web UI получил постоянную навигацию по разделам: обзор, интернет, домашняя сеть, Wi-Fi, система и обслуживание.
- Главная страница роутера теперь показывает краткое состояние сети, а не все данные и команды одновременно.
- Диагностика, capabilities, история команд и raw telemetry перенесены в отдельный служебный раздел.
- Список роутеров заменён на компактный список сетей с понятным статусом и временем последней связи.
- История команд сокращена до рабочего представления; payload и result раскрываются только по запросу.
- Адаптивный Web UI проверен на desktop и mobile без горизонтального переполнения.
- Android получил новый обзор роутера: интернет, клиенты, Wi-Fi и состояние системы.
- Управление агентом в Android перенесено с главного экрана в раздел «Система».
- Нижняя вкладка выбранного роутера теперь называется «Обзор».
- Исправлено возвращение Web UI в текущий раздел после отправки команды.
- GitHub Release публикуется как `Latest`, чтобы Android-проверка обновлений видела актуальную сборку.

## Обновление

Сервер: выполните redeploy образа `ghcr.io/shurshick/wrtmonitor:latest`. PostgreSQL volume сохраняется.

Android: установите `wrtmonitor-android-v0.2.0-rc2-debug.apk` поверх предыдущей версии. `versionCode` увеличен до `27`.

OpenWrt agent функционально не менялся. Артефакт `rc2` опубликован для единой версии комплекта; обязательного обновления агента с `rc1` нет.
