# v0.14.0 — Mobile Pairing

Стабильный релиз безопасного подключения Android-приложения к WrtMonitor Server через одноразовый QR-код.

## Изменения

- В Web UI добавлено создание QR со сроком действия 10 минут, обратным отсчётом и ручным отзывом.
- QR содержит только публичный адрес сервера и криптографически стойкий одноразовый pairing token.
- Сервер хранит только hash токена, блокирует повторное использование и ограничивает перебор через PostgreSQL rate limiting.
- После обмена создаётся обычная управляемая мобильная access/refresh-сессия; её можно отозвать в разделе «Аккаунт».
- Android получил сканер QR, проверку формата v1, подтверждение адреса сервера и отдельные ошибки для истёкшего, использованного и отозванного кода.
- Ручное подключение по адресу и паролю сохранено.
- Plaintext fallback для Android-токенов удалён: используется только `EncryptedSharedPreferences`.
- HTTP допускается только для локального адреса при `WRTMONITOR_ALLOW_INSECURE_LOCAL=true`.

## Обновление

Сервер обновляется через TrueNAS redeploy: **Apps -> WrtMonitor -> Edit -> Save**. Релиз содержит миграцию PostgreSQL `0007_mobile_pairing`; Alembic применит её автоматически при запуске контейнера. Данные роутеров и владельца сохраняются.

Агент получит версию metadata `0.14.0` обычным автообновлением. Capability schema остаётся `13`, переустановка агента не требуется.
