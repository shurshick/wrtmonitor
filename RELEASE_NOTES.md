# v0.14.1 — Client Presence

Релиз исправляет ложные статусы клиентов домашней сети и вводит проверяемую модель присутствия.

## Изменения

- Статусы клиентов разделены на «В сети», «Недавно» и «Не в сети».
- DHCP-аренды, `PERMANENT` и `NOARP` больше не считаются доказательством активности.
- Присутствие подтверждают активная Wi-Fi station, neighbour `REACHABLE`/`DELAY`/`PROBE` или рост клиентских RX/TX-счётчиков.
- `STALE` переводит клиента в «Недавно» и не продлевает этот статус бесконечно.
- `FAILED` и `INCOMPLETE` немедленно отмечают клиента отключённым.
- При остановке агента подтверждение автоматически проходит состояния `online → recent → offline`.
- Web UI и Android получили одинаковые счётчики, фильтры, источник статуса и время последнего подтверждения.
- Добавлена миграция PostgreSQL `0008_client_presence`; Android `versionCode` повышен до `57`.

## Обновление

Сервер обновляется через TrueNAS redeploy: **Apps → WrtMonitor → Edit → Save**. Контейнер автоматически применит миграцию `0008_client_presence`; данные владельца, роутеров и история клиентов сохраняются.

Агент получит metadata `0.14.1` обычным автообновлением. Capability schema остаётся `13`, переустановка не требуется.
