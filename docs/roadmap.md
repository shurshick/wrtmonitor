# Roadmap

Этот документ фиксирует будущий функционал, который не входит в текущую тестовую версию, но должен быть учтён при развитии проекта.

## Локальный поиск и подключение OpenWrt-роутера из Android

Статус: будущий функционал.

Цель: упростить первичное подключение роутера. Пользователь открывает Android-приложение, нажимает кнопку поиска, приложение находит OpenWrt-роутер в локальной сети и предлагает подключить его к уже работающему серверу `wrtmonitor`.

Исходные условия:

- сервер развёрнут и работает;
- Android-приложение установлено и подключено к серверу;
- OpenWrt-агент установлен на роутер, но может быть ещё не привязан к серверу.

Ожидаемый сценарий:

1. Пользователь нажимает в Android кнопку поиска.
2. Приложение сканирует локальную сеть телефона.
3. Найденные роутеры показываются списком.
4. Пользователь выбирает роутер и подтверждает подключение.
5. Приложение авторизуется на сервере логином и паролем владельца.
6. Сервер выдаёт device token через `/api/v1/devices/provision`.
7. Android передаёт роутеру адрес сервера и device token через локальный setup endpoint агента.
8. Агент сохраняет настройки, перезапускается и начинает работать через сервер.

Необходимые изменения:

- добавить в OpenWrt-агент локальный setup/discovery endpoint;
- добавить в Android экран поиска, список найденных устройств и flow подключения;
- добавить на сервер аудит события подключения роутера через мобильное приложение;
- продумать защиту локальной настройки: одноразовый PIN, временное окно настройки или явное включение setup-режима на роутере.

Оценка сложности:

- MVP: 3-5 рабочих дней;
- надёжная версия с нормальной безопасностью и UX: 1-2 недели.

Риски и ограничения:

- Android ограничивает сетевое сканирование и доступ к Wi-Fi-информации;
- телефон должен быть подключён к той же локальной сети, где находится роутер;
- подсеть может отличаться от `/24`, её нужно определять автоматически;
- локальный setup endpoint нельзя оставлять открытым без защиты.
