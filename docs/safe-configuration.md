# Безопасное изменение конфигурации

Начиная с `v0.4.0`, сервер не отправляет команды изменения UCI старому агенту без capability `config.transaction`.

## Жизненный цикл

1. Web UI или Android запрашивает `POST /api/v1/devices/{device_id}/commands/preview`.
2. Сервер нормализует payload, проверяет адреса и конфликты, возвращает diff и предупреждения.
3. При создании команды сервер добавляет служебный блок `_transaction` с ID, списком UCI-файлов и timeout.
4. Агент проверяет UCI-секции и свободное место, затем копирует только затрагиваемые файлы из `/etc/config`.
5. После применения сетевой команды агент переводит её в `running` и проверяет доступность сервера.
6. При восстановленной связи команда завершается как `success`; при timeout исходные файлы возвращаются и сервисы перезапускаются.

Стандартный timeout проверки связи - 90 секунд. Агент ограничивает полученное значение диапазоном 30-180 секунд.

## Защищённые команды

- Wi-Fi: состояние, SSID, пароль, канал, страна и гостевая сеть;
- WAN и LAN;
- DHCP, DNS и статические выдачи;
- проброс портов и блокировка клиента;
- hostname, timezone и NTP.

Секреты в preview и истории заменяются на `********`.

## Состояния результата

- `prepared` - резервная копия создана;
- `verifying` - конфигурация применена, агент проверяет связь;
- `confirmed` - связь восстановлена, изменение подтверждено;
- `rolled_back` - исходная конфигурация восстановлена;
- `rollback_failed` - восстановление завершилось ошибкой и требует локальной проверки роутера.
