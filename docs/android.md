# Android-приложение

Android-приложение - основной пользовательский интерфейс `WrtMonitor`.

Текущая версия: `v0.11.0`.

## Установка

1. Откройте релиз `v0.11.0`.
2. Скачайте APK:

   ```text
   wrtmonitor-android-v0.11.0.apk
   ```

3. Установите APK на Android-устройство.
4. При первом запуске укажите адрес сервера:

   ```text
   https://monitor.example.ru
   ```

5. Войдите логином и паролем администратора, созданного на `/setup`.

## Что уже работает

- ввод адреса сервера при первом запуске;
- вход администратора через backend API;
- безопасное хранение access/refresh token через `EncryptedSharedPreferences`;
- автоматическое обновление истёкшей сессии без повторного ввода пароля;
- список роутеров;
- компактные карточки роутеров со статусом и меню действий;
- пять основных вкладок после выбора роутера: обзор, клиенты, Wi-Fi, интернет и «Ещё»;
- отдельный каталог сетевых правил, VPN, системы и настроек приложения;
- иерархическая навигация назад без потери выбранного роутера;
- обзор роутера с состоянием интернета, клиентов, Wi-Fi и системы;
- рабочие разделы со сводкой, основными действиями и раскрываемыми расширенными настройками;
- telemetry, клиенты, Wi-Fi, интернет, сетевые правила, VPN, система и agent status;
- статические DHCP-выдачи;
- несколько SSID, radio, канал, ширина, мощность, расписание, roaming и Mesh;
- переподключение интерфейсов и перезапуск сети;
- hostname и перезапуск ключевых OpenWrt-сервисов;
- WAN DHCP/static/PPPoE, LAN, DNS и проброс портов;
- DHCP pool, блокировка клиентов и гостевой Wi-Fi;
- постоянный реестр клиентов, профили, расписания и приоритет трафика;
- история RX/TX по клиентам и общая настройка SQM;
- timezone и NTP;
- обслуживание агента и compact capabilities summary в разделе системы;
- capability-aware управляющие действия;
- diagnostics;
- экран "О приложении";
- dark Material/Flat тема.
- IPv6, Multi-WAN, маршруты, DDNS, UPnP и расширенные правила firewall.
- WireGuard, OpenVPN client, policy routing и VPN telemetry.

## Обновление APK

APK подписывается отдельным production-ключом WrtMonitor. При переходе с `v0.9.0` и более раннего debug APK нужна одна чистая установка: Android не разрешает обновление приложения с другим ключом подписи. Начиная с `v0.10.0` последующие версии устанавливаются поверх без удаления, а GitHub Release больше не публикует debug APK.

Источник версии для Android-сборки теперь один: `VERSION` и `VERSION_CODE` в корне репозитория. Это убирает старую проблему, когда имя релизного APK и фактическая версия приложения расходились.

## Ограничения текущей версии

- команды выполняются через очередь и приходят при следующем polling agent;
- локальный поиск роутера по LAN пока вынесен в будущий этап;
- отдельные команды доступны только при наличии соответствующей capability на конкретном OpenWrt-роутере.
