# Android-приложение

Android-приложение — основной пользовательский интерфейс `wrtmonitor`.

Текущая версия: `v0.1.0-test.15`.

## Установка

1. Откройте релиз:
   [v0.1.0-test.15](https://github.com/shurshick/wrtmonitor/releases/tag/v0.1.0-test.15)

2. Скачайте APK:

   ```text
   wrtmonitor-android-v0.1.0-test.15-debug.apk
   ```

3. Установите APK на Android-устройство.

4. При первом запуске укажите адрес сервера:

   ```text
   https://monitor.example.ru
   ```

5. Войдите логином и паролем администратора, созданного на `/setup`.

## Что уже работает

- ввод адреса сервера при первом запуске;
- вход администратора через backend API;
- список роутеров с сервера;
- экран устройства;
- API последней телеметрии;
- отображение статуса, uptime, load, памяти, Wi-Fi radios, stale telemetry;
- переключение русского/английского через системный язык Android.

## Экран устройства

Экран устройства показывает:

- online/offline статус;
- последнюю связь;
- возраст telemetry;
- предупреждение, если telemetry устарела;
- uptime;
- load average;
- доступную/общую память;
- Wi-Fi radios и SSID;
- наличие network telemetry.

Raw telemetry не является основным экраном. Она доступна по кнопке `Показать raw telemetry`.

## Обновление APK

1. Скачайте APK новой версии из релиза.
2. Установите поверх предыдущей версии.
3. Если приложение не может войти, смените сервер в настройках или очистите данные приложения и войдите снова.

Начиная с `v0.1.0-test.12`, debug APK подписывается стабильным тестовым ключом и имеет повышаемый `versionCode`, поэтому следующие тестовые APK должны ставиться поверх предыдущих.

Если на телефоне уже стоит APK из `v0.1.0-test.11` или старее, Android может отказать в обновлении из-за другой debug-подписи. В этом случае один раз удалите старое приложение и установите `v0.1.0-test.12`; дальнейшие обновления должны ставиться поверх.
