# Android-приложение

Android-приложение - основной пользовательский интерфейс `WrtMonitor`.

Текущая версия: `v0.2.0-rc1`.

## Установка

1. Откройте релиз `v0.2.0-rc1`.
2. Скачайте APK:

   ```text
   wrtmonitor-android-v0.2.0-rc1-debug.apk
   ```

3. Установите APK на Android-устройство.
4. При первом запуске укажите адрес сервера:

   ```text
   https://monitor.example.ru
   ```

5. Войдите логином и паролем администратора, созданного на `/setup`.

## Что уже работает

- ввод адреса сервера при первом запуске;
- вход администратора через backend API;
- безопасное хранение сессии и access token через `EncryptedSharedPreferences`;
- список роутеров;
- экран устройства;
- telemetry, клиенты, Wi-Fi, сеть, система и agent status;
- статические DHCP-выдачи;
- Wi-Fi канал, регион, SSID, пароль и состояние;
- переподключение интерфейсов и перезапуск сети;
- hostname и перезапуск ключевых OpenWrt-сервисов;
- compact capabilities summary на экране устройства;
- capability-aware управляющие действия;
- diagnostics;
- экран "О приложении";
- dark Material/Flat тема.

## Обновление APK

Debug APK подписывается стабильным тестовым ключом и имеет увеличиваемый `versionCode`, поэтому `v0.2.0-rc1` ставится поверх более ранних RC-версий без удаления приложения.

Источник версии для Android-сборки теперь один: `VERSION` и `VERSION_CODE` в корне репозитория. Это убирает старую проблему, когда имя релизного APK и фактическая версия приложения расходились.

## Ограничения текущей версии

- команды выполняются через очередь и приходят при следующем polling agent;
- локальный поиск роутера по LAN пока вынесен в будущий этап;
- production-signed APK ещё не входит в релизный контур.
